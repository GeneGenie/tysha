# Тишина

> Приложение называется **Тишина** (имя на домашнем экране и заголовок).
> Внутреннее имя проекта/таргета в Xcode осталось `BreathOS` — на пользователя
> это никак не влияет.

Нативное iOS-приложение (SwiftUI, iOS 17+) для дыхательной практики гипоксического
типа (схема Вима Хофа): серии быстрых дыханий → длинная задержка на выдохе →
восстановительный вдох с задержкой, несколько раундов. Фон — Metal-шейдер, звук из
mp3, тактильная отдача (Taptic Engine). Интерфейс — русский, тёмная тема. Без сети,
аккаунтов, аналитики. Только локальный запуск с Mac на iPhone по кабелю.

## Структура проекта

```
BreathOS.xcodeproj          — проект Xcode (готов к открытию)
BreathOS/
  BreathOSApp.swift         — точка входа, RootView, навигация настройки ↔ сессия
  BreathModels.swift        — фазы, BreathSettings (+UserDefaults), раскладка в очередь фаз
  BreathEngine.swift        — движок: CADisplayLink, гаптика, аудио, публикуемое состояние
  AudioConfig.swift         — настройки фоновой музыки (старт/громкость) для разработчика
  SettingsView.swift        — Экран 1: настройки
  SessionView.swift         — Экран 2: сессия (шейдер + оверлеи + «Готово»)
  BreathShaderView.swift    — TimelineView + Shader, маппинг состояния в юниформы
  Breath.metal              — шейдеры (FBM, домен-уорпинг, вспышка, виньетка, кроссфейд)
  Assets.xcassets           — AppIcon (пустой) + AccentColor
  Resources/                — звуки фаз и фоновая музыка
    inhale.mp3              — звук вдоха   (копия resources/breath_in.mp3)
    exhale.mp3              — звук выдоха  (копия resources/breath_out.mp3)
    exhale_long.mp3         — растянутый до ~4 с выдох (закрывает раунд)
    bg_music_start_from_45sec.mp3 — фоновая музыка (играет тихо под фазами)
```

> Файлов `hold_in.mp3` и `hold_out.mp3` нет — соответствующие фазы (задержки)
> проигрываются **молча**, приложение не падает. Чтобы добавить им звук, положите
> mp3 с этими именами в `BreathOS/Resources/` и добавьте их в target (см. ниже).

## Как запустить (Mac → iPhone по кабелю)

> ⚠️ Нужен **полноценный Xcode** (App Store), не только Command Line Tools.
> Проверить: `xcode-select -p` должно указывать на `…/Xcode.app/…`. Если нет —
> установите Xcode и выполните `sudo xcode-select -s /Applications/Xcode.app`.

1. **Откройте проект.** Двойной клик по `BreathOS.xcodeproj` (или `File ▸ Open`).
   Если открыть проект таким образом не удаётся, см. раздел «Если проект не
   открывается» ниже — там сборка вручную за пару минут.
2. **Подключите iPhone кабелем** к Mac, разблокируйте, при запросе нажмите
   «Доверять этому компьютеру». В Xcode сверху (рядом с кнопкой Run) выберите свой
   iPhone как run target вместо симулятора.
3. **Подпись.** Выберите target **BreathOS** ▸ вкладка **Signing & Capabilities**:
   - включите **Automatically manage signing**;
   - в **Team** выберите свой Apple ID (Personal Team). Если его нет — `Xcode ▸
     Settings ▸ Accounts ▸ +` и войдите бесплатным Apple ID;
   - при необходимости поменяйте **Bundle Identifier** на свой уникальный
     (например `com.вашеимя.breathos`) — бесплатной подписи нужен уникальный id.
4. **Запуск.** Нажмите **Run** (▶) или `Cmd+R`. Xcode соберёт и поставит приложение
   на телефон.
5. **Доверьте сертификат разработчика на телефоне** (при первом запуске iOS
   откажется открыть приложение):
   `Settings ▸ General ▸ VPN & Device Management` → выберите свой профиль
   разработчика → **Trust**. После этого запустите приложение с экрана iPhone.

### Звук и mp3

- Звуки уже добавлены в проект (Resources в Copy Bundle Resources).
- Чтобы заменить/добавить: перетащите mp3 в группу **Resources** в навигаторе Xcode,
  в диалоге отметьте **Copy items if needed** и target **BreathOS**. Либо вручную:
  target ▸ **Build Phases ▸ Copy Bundle Resources ▸ +**.

### Куда класть ресурсы (иконка, сплэш, звуки)

Все мок-ресурсы заменяются **файлами с теми же именами** — ничего в Xcode менять
не нужно, просто перезапишите файл и пересоберите:

| Что | Куда | Формат |
|---|---|---|
| Иконка приложения | `BreathOS/Assets.xcassets/AppIcon.appiconset/AppIcon.png` | PNG **1024×1024**, непрозрачный (без альфы) |
| Лого сплэш-экрана 1x | `BreathOS/Assets.xcassets/LaunchLogo.imageset/launch-logo.png` | PNG 180×180, прозрачный фон |
| Лого сплэш-экрана 2x | `BreathOS/Assets.xcassets/LaunchLogo.imageset/launch-logo@2x.png` | PNG 360×360 |
| Лого сплэш-экрана 3x | `BreathOS/Assets.xcassets/LaunchLogo.imageset/launch-logo@3x.png` | PNG 540×540 |
| Цвет фона сплэша | `BreathOS/Assets.xcassets/LaunchBackground.colorset/Contents.json` | правится RGB в самом json |
| Звуки | `BreathOS/Resources/*.mp3` | `inhale`, `exhale`, `exhale_long`, `hold_in`, `hold_out`, `bg_music_start_from_45sec` |

Нюансы:
- Сплэш в iOS — это **одна** статичная картинка по центру + цвет фона
  (`Info.plist → UILaunchScreen`). Анимаций и нескольких изображений система не
  поддерживает. Лого рисуется в натуральную величину: 540 px @3x = 180 pt на экране.
  Хотите крупнее — кладите PNG большего размера с теми же именами.
- iOS агрессивно кэширует сплэш: после замены картинки удалите приложение с
  телефона и поставьте заново (или перезагрузите телефон), иначе можете видеть старую.
- Если поменяете **имена** файлов — обновите соответствующий `Contents.json` рядом.
- Новые звуки с другими именами надо добавить в Xcode: перетащить в группу
  Resources с галочкой target BreathOS (Build Phases → Copy Bundle Resources).

### Аудио (для разработчика)

Параметры в `AudioConfig.swift`:
- `backgroundMusicStartSec` — с какой секунды начинать (и зацикливать). По умолчанию **90**.
- `backgroundMusicVolume` — громкость музыки 0…1 (по умолчанию 0.25, тихо под фазами).
- `breathVolume` — громкость дыхательных звуков 0…1 (по умолчанию **0.6**, чтобы музыка оставалась слышна).
- `backgroundMusicFile` — имя файла без расширения.

Звук играет только при включённом тумблере **Звук**.

## ⏳ Важно про бесплатную подпись

Приложение, подписанное **бесплатным Apple ID**, работает на телефоне **7 дней**,
после чего его нужно переустановить через Xcode (повторный Run). Платный **Apple
Developer Program ($99/год)** снимает ограничение и даёт раздачу через TestFlight.

## Если проект не открывается

Файл проекта собран вручную. Если Xcode выдаёт ошибку при открытии
`BreathOS.xcodeproj`, создайте проект сами за пару минут:

1. `File ▸ New ▸ Project… ▸ iOS ▸ App`. Product Name: **BreathOS**, Interface:
   **SwiftUI**, Language: **Swift**, Minimum Deployments: **iOS 17.0**. Сохраните
   во временную папку.
2. Удалите автосозданные `ContentView.swift` и `<Name>App.swift`.
3. Перетащите в проект все файлы из папки `BreathOS/` этого репозитория
   (`*.swift`, `Breath.metal`, `Assets.xcassets`) и папку `Resources/` —
   с галочками **Copy items if needed** и target **BreathOS**.
4. Дальше — пункты 2–5 из «Как запустить».

## Заметки по доводке шейдера

Фрактальный/визуальный фон обычно требует 2–4 итераций правок. Кандидаты на тюнинг
в `Breath.metal`:
- скорость потока — множители `t * 0.10…` в `warpedField` и `driftScale`;
- насыщенность голубого — `bluePalette` (`deep`/`light`) и контраст `smoothstep(0.15, 0.95, f)`;
- резкость/мягкость вспышки восстановительного вдоха — `renderFlash` (`maxR`, `smoothstep(0.0, 0.3, …)`, `radius * 0.15`);
- чёрные волны длинной задержки — `renderHoldOut`: скорость наползания (`t * 0.45`),
  «волнистость» границы (`(n - 0.5) * 0.35`), крупность волн (`cn * 3.5`),
  мягкость кромки (`lightR - 0.22`);
- крупность FBM — множитель `uv * 3.0` в `warpedField`.

Поведение фона: во время серии дыханий фон **не переключается** (ровный голубой
фрактал); вспышка — только на восстановительном вдохе; на длинной задержке чёрные
процедурные волны сходятся со всех сторон сразу с начала фазы. Каждый раунд
закрывается длинным выдохом ~4 с (`exhale_long.mp3`; если файла нет — обычный
выдох замедляется программно).

Также для первого демо разумно поставить меньшие задержки (например 30/45/60) — это
делается прямо на экране настроек в списке «Задержки на выдохе по раундам».
