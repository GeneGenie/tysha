# ТЗ: iOS-приложение «BreathOS» (дыхательные практики)

## Общее

Нативное iOS-приложение на **SwiftUI**. Минимальная версия **iOS 17.0**. Язык интерфейса — **русский**. Тёмная тема (`preferredColorScheme(.dark)`).

Цель: вести пользователя по дыхательной практике гипоксического типа (схема Вима Хофа) — серия быстрых дыханий, затем длинная задержка на выдохе, затем восстановительный вдох с задержкой, и так несколько раундов с растущей длительностью задержки. Визуализация на Metal-шейдерах, звук из mp3, тактильная отдача (гаптика = вибро-импульсы Taptic Engine).

Создай **полный Xcode-проект**, который открывается и запускается на физическом iPhone через бесплатный Apple ID (подпись на 7 дней — норм для демо). Без сети, аккаунтов, аналитики и App Store-сборки. Только локальный запуск с Mac на iPhone по кабелю.

---

## Модель сессии

Сессия состоит из **раундов**. Каждый раунд проходит по этапам:

1. **Серия дыханий** — `breathsPerRound` повторов: вдох `inhaleSec` → выдох `exhaleSec`, **без задержек между дыханиями**.
2. **Длинная задержка на выдохе** — одна на раунд, длительность зависит от номера раунда (берётся из массива `holdOutByRound`).
3. **Восстановительный вдох + задержка на вдохе** — глубокий вдох, затем держим `recoveryHoldSec` секунд.
4. Переход к следующему раунду.

Набор фаз для движка и шейдера (индекс шейдера в скобках):

| Фаза | Индекс для шейдера | Где используется |
|---|---|---|
| `inhale` | 0 | вдох в серии; восстановительный вдох |
| `holdIn` | 1 | восстановительная задержка на вдохе |
| `exhale` | 2 | выдох в серии |
| `holdOut` | 3 | длинная задержка на выдохе |

Порядок и длительности фаз задаёт модель раунда, а не фиксированный цикл. Фазы с длительностью 0 пропускаются.

## Экран 1. Настройки (стартовый)

Ползунки / поля:

- **Вдох (серия):** 0.5–5 с, шаг 0.1. По умолчанию **2.0**.
- **Выдох (серия):** 0.5–5 с, шаг 0.1. По умолчанию **1.6**.
- **Дыханий в раунде:** 20–40, шаг 1. По умолчанию **30**.
- **Восстановительная задержка на вдохе:** 5–30 с, шаг 1. По умолчанию **10**.
- **Количество раундов:** 1–8, шаг 1. По умолчанию **5**.

**Задержки на выдохе по раундам** — редактируемый список:

- При изменении числа раундов список подгоняется по длине.
- Значения по умолчанию (секунды): **60, 90, 120, 150, 180**.
- Если раундов больше 5 — недостающие заполнять последним значением (180).
- Если меньше — брать первые N значений.
- Каждое значение редактируется отдельным полем/степпером, диапазон 15–300 с, шаг 5.

Тумблеры: **Звук** (вкл/выкл), **Вибрация** (вкл/выкл).

Строка-предупреждение (обязательна, держать также на экране сессии):

> Выполняйте лёжа. Не в воде, не за рулём, не в одиночку в первый раз. При сжатии в груди, потемнении в глазах или онемении — немедленно прекратите.

Кнопка **«Начать»**. Все настройки сохраняются между запусками (UserDefaults).

## Экран 2. Сессия

Полноэкранный шейдер-фон (см. «Визуализация»). Поверх него:

- **Сверху:** счётчик «Раунд X/Y». Кнопка «Выход».
- Во время серии дыханий — дополнительно «Дыхание N/breathsPerRound».
- **Под подписями по центру:** название текущей фазы и крупный отсчёт секунд («часики»).
  - Для фазы `holdOut` (длинная задержка) — **прямой счёт вверх** (00:00 → растёт), так нагляднее «сколько продержался».
  - Для остальных фаз — **обратный отсчёт** до конца фазы.
- Текст должен оставаться читаемым на любом фоне (тень или полупрозрачная подложка — критично на светлой вспышке вдоха).
- По завершении всех раундов — экран **«Готово»** с кнопкой «Ещё раз».

## Визуализация (Metal-шейдер, фон на весь экран)

Рендер через SwiftUI `Shader` (`colorEffect` / `layerEffect`) с `.metal`-файлом. Анимация по непрерывному `time` через `TimelineView(.animation)`.

**Юниформы шейдера:**

- `time` — секунды, непрерывно
- `phase` — int 0–3 (0=вдох, 1=задержка-вдох, 2=выдох, 3=задержка-выдох)
- `phaseProgress` — 0…1 внутри текущей фазы
- `resolution` — размер в пикселях

**Поведение по фазам:**

- **Вдох (особенно восстановительный после длинной задержки):** резкая вспышка света из центра. Из середины расходится яркое пятно (радиальный градиент белый → прозрачный) с сильным размытием краёв; радиус растёт от 0 к полному экрану за первые ~30% фазы, затем держится. Начало намеренно резкое.
- **Выдох:** голубая фрактальная анимация в движении — анимированный **FBM-шум** (4–5 октав, hash-based value noise, без текстур) с **домен-уорпингом**, текстура «течёт». Палитра голубая (от `#4DB3F2` к глубокому сине-бирюзовому). Поток медленный, медитативный.
- **Задержка на вдохе:** тот же фрактал, что на выдохе, но дрейф ×0.2 (медленное парение), яркость не меняется.
- **Длинная задержка на выдохе:** белый фон постепенно темнеет **от краёв к центру**. Виньетка, тёмное кольцо стягивается внутрь: при `phaseProgress` 0 светлый почти весь экран (тёмная узкая кромка по периметру), при `phaseProgress` 1 тёмное почти всё, светлая точка в центре. Плавно, без рывков.

**Переходы между фазами:** кроссфейд по альфе ~0.5 с. Исключение — вспышка вдоха, она резкая.

**Требования к шейдеру:** целевой 60 FPS; не более 5 октав; вычисления во `float` (не `double`); FBM считать в самом шейдере без текстур.

## Гаптика (тактильная отдача)

- На **каждой смене фазы** — `UIImpactFeedbackGenerator(style: .heavy)`, `.impactOccurred()`.
- На старте и на завершении **длинной задержки на выдохе** — дополнительный мягкий импульс (`.light` или `.medium`).
- На завершении **раунда** и всей **сессии** — `UINotificationFeedbackGenerator`, `.success`.
- Если тумблер «Вибрация» выключен — гаптики нет.
- Генераторы вызывать `prepare()` заранее.

---

## Звук

- mp3-файлы в бандле, по одному на фазу. Имена: `inhale.mp3`, `hold_in.mp3`, `exhale.mp3`, `hold_out.mp3`.
- Звук фазы проигрывается в момент **начала** фазы.
- Если файла нет — приложение работает молча, без краша.
- `AVAudioSession` режим `.playback`, опция `.mixWithOthers`. Плееры `prepareToPlay()` заранее.
- Если тумблер «Звук» выключен — звука нет.

---

## Тайминг и системное

- Точный таймер на **`CADisplayLink`**, отсчёт от `CACurrentMediaTime()` (не `Timer`).
- На время сессии экран **не гаснет**: `UIApplication.shared.isIdleTimerDisabled = true`, вернуть `false` при выходе/завершении.
  - Важно: в фоне с погашенным экраном iOS глушит и звук, и гаптику — поэтому держим экран активным всю сессию.
- `preferredColorScheme(.dark)`.

## Структура данных

```swift
struct BreathSettings: Codable {
    var inhaleSec: Double = 2.0
    var exhaleSec: Double = 1.6
    var breathsPerRound: Int = 30           // 20...40
    var recoveryHoldSec: Double = 10        // задержка на вдохе после гипоксии
    var rounds: Int = 5                     // 1...8
    var holdOutByRound: [Double] = [60, 90, 120, 150, 180]  // длина = rounds
    var soundEnabled = true
    var hapticsEnabled = true
}
```

Движок при старте раскладывает настройки в плоскую очередь фаз `[(phase, duration)]` и отыгрывает её по `CADisplayLink`. Публикует наружу: текущую `phase`, `phaseProgress`, отсчёт времени, номер раунда, номер дыхания в серии.

---

## Структура файлов (предложение)

- `BreathModels.swift` — enum фаз (индекс для шейдера, имя аудио, цвет), структура `BreathSettings` + сохранение/загрузка в UserDefaults, логика раскладки раундов в очередь фаз.
- `BreathEngine.swift` — движок сессии: очередь фаз, переходы, `CADisplayLink`, гаптика, аудио, публикуемые свойства состояния.
- `BreathShaderView.swift` — `TimelineView` + `Shader`-фон, маппинг состояния движка в юниформы.
- `Breath.metal` — шейдеры (FBM, домен-уорпинг, вспышка из центра, виньетка от краёв, кроссфейд фаз).
- `BreathOSApp.swift` — точка входа, экран настроек, экран сессии.

## Заметки по доводке (ожидаемые итерации)

Фрактальный шейдер «красиво» с первого прогона почти не выходит. Закладывай 2–4 итерации правок вида: «поток медленнее», «голубой приглушённее, менее кислотный», «вспышка мягче / резче», «виньетка стягивается плавнее», «FBM крупнее/мельче». Это нормальная часть процесса.

Для первого демо разумно поставить скромные задержки (например 30 / 45 / 60 вместо 60…180) и довести до больших значений отдельно — 180 секунд выдерживают только подготовленные.
